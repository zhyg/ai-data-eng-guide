# 导学 06：把前 5 节学的全串起来，做一次完整演练

## 一句话理解

这一节没有新概念，就是动手把前面学的串成一次"别人也能照着复现"的实验。

## 你要达成的 6 件事

1. 在本机起一套数据环境
2. 建好 4 张表的"骨架"
3. 至少往 1 张青铜表 + 1 张白银表里写入数据
4. 看到完整的"状态证据链"（快照、历史、文件、元数据）
5. 演示一次"加一列"的字段演进
6. 输出一份基线报告

## 七个步骤逐个说

### 第 1 步：起环境
用一行 `docker compose up -d` 把数据库、对象存储都拉起来。
**怎么算通过**：能打开 MinIO 控制台，能连上数据库，桶（bucket）已经建好。

### 第 2 步：建骨架
告诉账本"我要建这 4 张表"，但表里还没数据。
**怎么算通过**：列出表名能看到 4 张。

### 第 3 步：写数据
把上周的工单数据 / 文档读进来，按映射规则转换，写入表。
至少跑通 1 个青铜表 + 1 个白银表。

### 第 4 步：生成快照
每次写入会自动产生一个快照编号。记下来：
- 快照编号
- 上一个快照是谁
- 是新增（append）还是覆盖（overwrite）
- 提交时间

### 第 5 步：查看状态
按这个顺序看：

```
快照 → 历史 → 文件 → 元数据 → 基线
```

每一层告诉你不同的事：

| 视图 | 告诉你什么 |
| :--- | :--- |
| snapshots | 一共有几个快照、每个是什么操作 |
| history | 谁在什么时候成为"当前版本" |
| files | 现在用了哪些文件、多大、怎么分布 |
| metadata_log | 说明书是怎么一步步演进的 |
| baseline | 把这些结果整理成可复现的报告 |

### 第 6 步：加一列
挑一张青铜表，加一列新字段。
然后再写一小批数据进来，验证：
- 新数据有新字段
- 旧快照仍然能按旧结构读出来

### 第 7 步：写基线报告
按上节课的模板写，至少包含：
- 环境说明
- 4 张表的状态数字
- 异常现象的解释
- 加列演示的记录

## 验收清单（每项都要打勾）

- [ ] 4 张表骨架在

## 常见踩坑（再提醒一次）

| 现象 | 先看哪里 |
| :--- | :--- |
| 账本加载失败 | 配置、环境变量 |
| MinIO 连不上 | 是不是容器名写成 localhost |
| 桶不存在 | 路径和权限 |
| 字段不匹配 | 回去对照 schema 定义文件 |
| 重跑数据重复 | 去重规则、重跑策略 |
| 看不到快照 | 写入是否真的执行了 |
| 旧状态读不到 | 是不是太早清掉了 |

## 最后要交的 4 份产物

| 文件 | 它的作用 |
| :--- | :--- |
| 运行手册 | 别人照着能跑起来 |
| 加列笔记 | 字段演进的记录 |
| 基线报告 | 表当前的状态成绩单 |
| 时间旅行笔记 | 回看旧状态的演示记录 |

## 一句话总结

> **实验做完不算合格，"别人能照着复现"才算合格。**

完成这一步，你就拥有了一份**能记得住、能回看、能解释**的数据底座，
接下来才可以放心地进入下一周的内容（数据加工和语义层）。
