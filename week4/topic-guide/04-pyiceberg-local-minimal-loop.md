# 导学 04：在自己电脑上跑通"最小闭环"

## 一句话理解

不上重型平台，用最轻的工具，在自己电脑上把"写入→形成快照→能回看"这一圈走通。

## 为什么不直接上"重武器"？

行业里有很多大块头工具（Spark、Hive、Trino……），但对初学者来说：

- 装起来麻烦
- 出错了不知道是哪一层的问题
- 真正的核心机制反而被遮住了

所以本周用最轻的三件套：

```diagram
╭─────────────╮    ╭────────────────────────╮    ╭──────────────╮
│ PyIceberg   │───▶│ PostgreSQL             │───▶│ MinIO        │
│ (Python库) │    │ (登记有哪些表、在哪里) │    │ (放数据文件) │
╰─────────────╯    ╰────────────────────────╯    ╰──────────────╯
   操作工具         只存"目录"，不存数据       真正存数据的地方
```

容易混淆：**PostgreSQL 里不存表数据，只存"表的目录信息"**（哪些表存在、说明书在哪）。
真正的数据文件都在 MinIO 里。

## 三层配置先分清楚

很多人一开始就被配置绕晕，其实只要分清三件事：

| 配置 | 它在管什么 |
| :--- | :--- |
| **Catalog（目录）** | 记录"哪些表存在，它们的说明书在哪" |
| **Warehouse（仓库）** | 数据文件和说明书统一放在这个根目录下 |
| **Table Location（表位置）** | 某张具体的表，在仓库里的子文件夹 |

## 一条铁规：不要硬编码！

像数据库地址、密码、存储端点这些，**全部走环境变量**。
否则改环境就得改代码，迟早出事。

## 写入一张表的 10 个动作

可以理解成"做菜流程"：

```diagram
1. 读配置        →  看食谱
2. 连账本        →  打开锅
3. 准备命名空间  →  分好碗
4. 确认表结构    →  对照菜谱
5. 读输入数据    →  备菜
6. 按映射转换    →  切配
7. 写入表        →  下锅
8. 生成快照      →  装盘
9. 查看状态      →  尝味道
10. 输出报告     →  写菜评
```

## 5 条"最低防线"

写入前一定要做的检查，别图省事：

- **空跑一次**：先看看计划是什么，不要直接污染表
- **明确映射**：源字段对哪个目标字段，写清楚
- **校验字段**：类型、是否允许空，写入前检查
- **去重逻辑**：重复输入要能识别
- **重跑安全**：跑两次结果应该一样

## 验收清单（做完打勾）

- [ ] 账本能正常加载
- [ ] 命名空间（bronze / silver）建好了
- [ ] 4 张表的结构都存在
- [ ] 至少 1 张青铜表 + 1 张白银表写入了数据
- [ ] 能看到快照、历史、文件这些记录
- [ ] 演示了一次"加一列"的字段演进

## 出问题怎么排查？

| 报错现象 | 先看哪里 |
| :--- | :--- |
| 账本加载失败 | 配置文件、环境变量 |
| MinIO 连不上 | 是不是把容器名写成 localhost 了 |
| 桶不存在 | 仓库根路径写对没 / 有没有权限 |
| 字段不匹配 | 回到表结构定义文件对一下 |
| 重跑出现重复 | 去重规则、重跑策略 |
| 没看到快照 | 写入到底有没有真的发生 |

## 4 条避坑提醒

- 把密码写死在代码里 → 走环境变量
- 表是空的也算成功 → 必须看到快照证据
- 一上来就套调度系统 → 本周重点是 Iceberg 本身
- PPT 里好看，代码跑不通 → 演示必须能复现

## 一句话总结

> **跑通最小闭环 = 写入成功 + 能看到快照 + 别人能复现。少了哪个都不算通。**
