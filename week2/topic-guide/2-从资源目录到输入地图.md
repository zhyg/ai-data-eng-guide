# 导学 02：发个 Excel 收数据，为什么 AI 团队还是不敢用？

## 一句话理解

给 AI 系统盘点数据，要给每份数据建档案，写清它的身份、负责人、来源和使用边界。光列一份清单数数有多少，下游用不起来。

## 用生活打比方

继续我们的海关口岸。口岸建好了，第一件事是给每一批进来的货登记造册。

海关要在档案里写清楚：这批货是谁的、从哪来、由谁负责、能不能合法入境、进来后流向哪个市场。普通日用品和冷链药品处理方式完全不同，所以登记的单位是每一种货，不是每一个集装箱。

## 关键概念

| 概念 | 大白话解释 |
| :--- | :--- |
| **资产盘点（Asset Inventory）** | 给每份输入数据建档案的过程。记的是这份数据在系统里的身份、谁负责、谁用它，不是 Excel 里写一行名字。 |
| **五要素法** | 一份合格的数据档案要回答五件事，分三组：身份与负责人、来源与证据、边界与消费方。少了任何一组，下游都会卡住。 |
| **身份与负责人（Identity）** | 唯一编号（asset_id）、数据类型、负责团队、生命周期。比如一份工单数据是滚动更新的，出问题先找 support-platform 团队。 |
| **来源与证据（Evidence）** | 内容指纹、来自哪个系统、有没有使用许可、最近一次更新时间。比如一篇帮助文档来自 docs 系统，许可等级是公开。 |
| **边界与消费方（Boundary）** | 敏感等级、谁有权限看、被哪些下游服务消费、坏了多久业务会受影响。比如工单的 PII 等级是 P2，被 RAG 和 KPI 两个服务消费。 |
| **四类输入资产** | 工单（结构化，最危险的是枚举字段）、文档（核心靠元数据，比如 page_no）、音频（PII 最容易卡住，要区分说话人）、视频（拆成音轨、OCR、关键帧三轨处理）。 |

## 避坑提醒

- **以为按系统名盘点就够了**：按 CRM、工单系统、客服 IM 分组，开发好对接。但同一个系统里不同对象含义不同，比如工单里的状态字段和评论字段，出问题的方式完全不一样，按系统分组没法精细管理。
- **以为按文件类型分类很完备**：按 PDF、表格、音视频分，看着齐全，却丢掉了负责人、生命周期、合规边界这些关键信息。这些信息决定了数据坏了找谁、能不能用。
- **以为让业务线自己填 Excel 最省事**：推动快，但各业务线字段名对不齐、口径打架、敏感边界没人定。最后 AI 团队拿到一堆表，还是不敢直接用。
- **以为文档和音视频可以先放一放**：这类数据在 Demo 阶段看着能跑，但在生产里全靠元数据撑着。盘点时被低估，上线后最先出问题。

## 一句话总结

> **盘点数据要给每份数据写清身份、来源和使用边界。这些字段后面每一个都会被某个服务读取，光数数有多少没有意义。**
