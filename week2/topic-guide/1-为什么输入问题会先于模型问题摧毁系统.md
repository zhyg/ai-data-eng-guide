# 导学 01：模型没动，回答为什么越来越歪？

## 一句话理解

AI 系统在生产里最危险的状态，是它还在正常运行、却高自信地一直答错，而问题往往出在喂进去的数据上。

## 用生活打比方

这一周我们用「海关口岸通关」来串起所有概念。

AI 系统就像一个口岸，每天有大量数据进来。如果口岸不设防，假证件、变质食品、违禁品都能混进来，入境时一切正常，几周后才集中爆发。这一周做的事，就是给 AI 系统建口岸，把输入当第一关来管。

## 关键概念

| 概念 | 大白话解释 |
| :--- | :--- |
| **输入控制面（Input Control Plane）** | 在数据进入系统之前先检查、先拦截的那一层。好比海关口岸，所有东西先过这一关再放行。 |
| **静默失败（Silent Failure）** | 系统没崩，接口照常返回，但答案一直是错的。比如假证件顺利入境，没人发现，问题在国内慢慢发酵。 |
| **语义漂移（Semantic Drift）** | 字段名没变、格式也对，但含义被上游悄悄改了。比如工单状态从 pending 改成了 in_progress，下游还按老意思算，KPI 全错。 |
| **证据断链（Provenance Break）** | 答案可能是对的，但说不出来自哪一页、哪一段。客户来纠错时，你连原文在哪都答不上来。 |
| **事实底线（Fact）** | 保证字段的含义没被偷改。比如「pending」被上游改成了「in_progress」，意思变了但格式没坏，就是事实底线没守住。 |
| **证据底线（Evidence）** | 保证每个答案都能回指到原始来源。比如能说清这句话来自第几页第几段，而不是只给一段文字。 |
| **边界底线（Boundary）** | 保证敏感数据按权限处理。比如客户电话号码在入库前就该脱敏或拦截，防止越权暴露。 |

## 避坑提醒

- **以为答错就是模型不够聪明**：Demo 阶段答错确实常是模型或提示词的问题。但在生产环境中，遇到异常表现，应优先核查输入数据的准确性与一致性。
- **以为 Schema 校验通过就没事了**：Schema 只回答格式像不像，回答不了含义变没变。字段没报错，不等于字段意思没变。语义漂移就是这样漏过去的。
- **以为答案对了就算合格**：在生产里，答得对但说不出来源的答案通常不合格。没有出处，就没法引用、没法审计、没法复盘 bad case。
- **以为这是数据治理的活**：这一周做的是把检查提前到入口，在数据进入系统之前就拦住问题。根因不在入口修，下游就会一直修表面症状。

## 一句话总结

> **系统还在跑不代表答案是对的。把输入当入国门的第一关来管，才能堵住后面的连锁错误。**
