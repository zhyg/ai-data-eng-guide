# 导学 06：怎么证明前五节搭的门禁真的管用？

## 一句话理解

这节是动手课，把盘点、合同、声明、放行、留证这五层接成一条能在本机跑起来的控制链，再用测试和试跑验证它拦得住坏数据。

## 用生活打比方

继续我们的海关口岸。前五节分别建好了登记册、报关单、通关标准、报关清单和闸机。这一节是开关前的全流程演练。

在沙盘上模拟一次通关：让模拟货物走过闸机，看合格货能放行、可疑货被隔离、违禁品被退回。演练结束还要故意夹带违禁品，验证闸机真的拦得住。

## 关键概念

| 概念 | 大白话解释 |
| :--- | :--- |
| **控制链（Control Chain）** | 从资产盘点到合同、声明、放行、证据，首尾相接组成一条链。这一周的产出是这条完整的链，不是几个孤立文件。 |
| **三张输入表** | 盘点层的三份底表：资源清单（有哪些数据源）、最小元数据（每条数据的身份和来源信息）、PII 矩阵（哪些字段敏感、该怎么处理）。 |
| **四类资产合同** | 工单、文档、音频、视频四种数据类型，共用一套合同骨架，再按各自特点扩展字段。比如音频要管角色分离和转写置信度，视频要管三轨拆分和帧时间戳。 |
| **三态决策** | 闸机的三种结果：通过（accept）、警告（warn）、隔离（quarantine）。配上证据报告才算闭合。 |
| **试跑（dry-run）** | 这一周只试跑，不真接生产数据。加载并校验清单，对每个资产做放行判断，打印报告并生成证据文件。 |
| **破坏性测试** | 故意制造坏数据（比如删掉一个必需字段），验证闸机确实会拒绝。光看正常数据能通过，证明不了门禁有用。 |

## 避坑提醒

- **以为这周是在写一堆零散脚本**：它是一条完整的输入控制链。单看某个文件没意义，要理解它们从盘点到证据怎么串起来。
- **以为测试只要验证好数据能通过**：还得验证坏数据被拦下来。有效数据不误伤、无效数据全拦截、清单本身正确，三件都测到才算完整。
- **以为跑通试跑就能看到端到端效果**：这周只搭控制面、做试跑，不直接接真实采集。整体效果要到第三周接上数据入湖才完整呈现。
- **以为试跑出了报告就结束**：报告要能说清每个资产是通过、警告还是隔离，以及具体原因。比如元数据缺失或校验和不匹配，这些细节是后续评审和治理的依据。

## 一句话总结

> **把五节课的产出接成一条能跑的控制链，再用试跑和破坏性测试证明它拦得住坏数据，这一周的输入控制面就完整了。**
