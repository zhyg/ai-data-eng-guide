
```bash

  bash -lc 'cd analytics && DBT_PROFILES_DIR=. dbt debug'
```

```bash
docker compose --profile tools run --rm devbox \
  bash -lc 'cd analytics && DBT_PROFILES_DIR=. dbt docs generate'
```

## dbt 分层落地全景

### Sources（sources.yml）

```yaml
sources:
  - name: omni_postgres
    description: "Week03/Week04 已经加载到 PostgreSQL 的结构化工单数据"
    database: "{{ env_var('POSTGRES_DB', 'omnisupport') }}"
    schema: public
    tables:
      - name: ticket_fact
        columns:
          - name: ticket_id
            tests: [not_null, unique]
          - name: created_at
            tests: [not_null]
      - name: customer_dim
      - name: ticket_comment_fact
      - name: knowledge_doc
```

### Staging（stg_tickets.sql）

整形输入，rename / cast / 枚举标准化 / 派生布尔。不做 group by、不做跨表 JOIN。

### Intermediate（int_support_cases.sql + int_ticket_activity_daily.sql）

- JOIN ticket + customer + comment
- 派生业务事实：first_response_minutes、backlog_age_days、is_first_resolution_proxy
- 一行还是 case 粒度，不做最终聚合

### Marts（三个核心产出）

| Mart | Grain | 物化 | 作用 |
| :--- | :--- | :--- | :--- |
| **support_case_mart** | 一个 support case | table | case-level 切片，故意不选 PII 字段 |
| **support_kpi_mart** | 指标 x 日期 x 维度 | table | 11 个 metric 的宽表→长表翻转（LATERAL VALUES） |
| **agent_tool_input_view** | 同 kpi_mart | view | Agent 唯一能读的对象，白名单字段 |

### 宽表→长表翻转关键技巧

```sql
cross join lateral (
    values
        ('ticket_count',      daily.ticket_count::numeric),
        ('p1_ticket_count',   daily.p1_ticket_count::numeric),
        ('sla_breach_count',  daily.sla_breach_count::numeric),
        ('escalation_count',  daily.escalation_count::numeric),
        -- ... 11 个指标全部一次性翻转
) as metrics(metric_name, metric_value)
where metrics.metric_value is not null
```

## 测试落地

### Generic Tests（schema.yml）

- ticket_id：not_null + unique
- metric_name：accepted_values 白名单
- metric_value：not_null

### Singular Tests（自定义 SQL）

| 测试 | 保护什么 |
| :--- | :--- |
| no_pii_columns_in_agent_tool_input_view.sql | 安全视图不能泄露 PII 字段 |
| ratio_metrics_between_0_and_1.sql | ratio 类指标必须在 [0, 1] 区间 |
| metric_values_non_negative.sql | count 类指标不能出现负值 |

## Registry 落地

`analytics/metric_registry_v1.yml` 是 runtime contract，包含：

- 11 个指标的完整定义（name / label / owner / type / formula / sensitivity / status）
- 全局配置：safe_view / max_window_days / allowed_dimensions / allowed_roles
- 版本管理：registry_version: 1.1.0

### Registry Validator

```bash
python analytics/scripts/validate_metric_registry.py --json
# 预期：valid=true, metric_count=11
```

## 受控查询工具落地

### Tool Contract

`contracts/tools/tools/query_support_kpis_v1.json`：JSON Schema strict + additionalProperties=false。
