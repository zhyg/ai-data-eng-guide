# 生成课程导学（topic-guide）

从 topic 目录下的详细课程内容，提炼出面向数据工程初学者的导学文件。

## 前置条件

需要两样输入：

1. **topic 目录**：每个主题一个 markdown 文件，包含完整的课程内容（核心目标、案例、技术概念、关键论点等）。
2. **index.txt**（可选）：课程目录，列出本周所有主题。如果没有 index.txt，按 topic 目录下的文件顺序处理。

## 输出结构

每个 topic 对应一个导学文件，保存到同级的 `topic-guide/` 目录下，文件名与 topic 文件一致。

## 导学模板

每篇导学严格遵循以下结构，不增不减：

```markdown
# 导学 NN：[用问题或判断做标题，不用名词并列]

## 一句话理解

[用一句话说清这节课解决什么问题。不用"不是 X 而是 Y"句式。]

## 用生活打比方

[同一周的所有导学共用一个类比场景，每篇从这个场景的不同环节切入。
第一篇需要先介绍场景，后续篇用"继续我们的 XX"衔接。]

## 关键概念

| 概念 | 大白话解释 |
| :--- | :--- |
| **概念名（英文原词）** | 用日常语言解释，避免用其他术语来解释术语 |

## 避坑提醒

- **以为 X 就是 Y**：[解释为什么这个认知是错的，给出正确理解]

## 一句话总结

> **[一句话收尾，平实直接，不升华不鸡汤。]**

[可选：一句话过渡到下一节课，说清下一步做什么。]
```

## 写作规则

### 受众

数据工程初学者，可能没有专业背景。写给"第一次接触这些概念的人"看。

### 语言要求

1. **简练**：能用短句说清的不用长句，能删的字就删。
2. **少用术语**：关键概念表格里集中解释术语，正文尽量用大白话。术语首次出现时括注英文，后续只用中文。
3. **去 AI 味**：参照下方"禁用模式"清单，写完后自查一遍。

### 内容要求

1. **避免真实案例**：不引用原始 topic 中的真实客户故事、项目经历或具体事故（WAR STORY 等）。避坑提醒用通用的工程经验总结，不提及具体团队、客户、行业或事故细节。
2. **避免具体账单数据**：不出现真实的成本金额、月账单对比等敏感数据。用"成本远高于""大幅降低"等通用表述替代。

### 禁用模式

| 模式 | 处理 |
|------|------|
| "不是 X，而是 Y" / "不是 X 而是 Y" 对比框架 | 直接说结论 |
| 破折号（U+2014 `—`） | 用逗号或句号 |
| 段末重述句（"这说明""可以看出""到这里"） | 删掉 |
| 升华句（把工程观察上升为人生道理） | 删掉或改成具体建议 |
| 讲解腔起手（"真拆开看""这背后是"） | 删掉，直接说 |
| "恰恰相反""更可怕的是" 等情绪渲染 | 平实表述 |
| "X 缺一不可" 工整并列总结 | 换成自然语气 |
| "先立住""归根到底" 劝告腔 / 升华开头 | 直接陈述 |
| "值得注意的是""综上所述" | 删掉 |
| "非常""极其" | 用"很" |
| 引号做强调（"翻译""提升"） | 去掉引号或改写 |

### 类比规则

1. 同一周的所有导学**共用一个类比场景**（比如"中央厨房给连锁餐厅供餐"）。
2. 每篇从这个场景的不同环节切入，保持连贯。
3. 第一篇介绍场景，后续篇用"继续我们的 XX"衔接。
4. 类比要和原始 topic 的核心概念对应准确，不能为了好听牺牲准确性。

### 关键概念表格规则

1. 只提取该 topic 的核心概念，不贪多。一般 5~8 个。
2. "大白话解释"列不能用其他术语来定义术语。
3. 解释要足够具体，带例子（"比如..."）。
4. 概念名加粗，英文原词用括号跟在中文后面。

### 避坑提醒规则

1. 从原始 topic 中提取初学者最容易犯的错误认知。
2. 格式统一为 `**以为 X 就是 Y**：[解释]`。
3. 一般 3~5 条，不贪多。
4. 每条要说清"错在哪、为什么错"，不只是说"别这样"。

## 工作流程

1. **读 topic 文件**：通读原始内容，标记核心概念、常见误区、关键论点。
2. **确定类比场景**：如果是一周的第一篇，选定类比场景；否则延续已有场景。
3. **写初稿**：按模板逐节填写。
4. **自查去 AI 味**：对照禁用模式清单，逐条检查并修改。
5. **交叉校验**：对比原始 topic，确认关键概念没有遗漏或曲解。
6. **写完一篇确认无误后再写下一篇**，避免批量生成后再回头改。

## 校验清单

每篇导学完成后，对照检查：

- [ ] 结构完整：一句话理解、类比、概念表、避坑、总结都有
- [ ] 类比和本周其他导学共用同一个场景
- [ ] 关键概念表格里的解释不依赖其他术语
- [ ] 避坑提醒来自原始 topic 的真实内容，不是凭空编的
- [ ] 没有出现禁用模式清单里的 AI 痕迹
- [ ] 没有引用真实案例、具体客户故事或敏感账单数据
- [ ] 原始 topic 的核心概念没有遗漏或曲解
