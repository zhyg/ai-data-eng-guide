# 导学 03：客户问"这答案哪来的"，你答得上来吗？

## 一句话理解

RAG 系统必须能让客户看到"答案来自哪份文件的哪一页哪一段"，答不上来就只是 Demo。

## 用生活打比方

继续我们的图书馆档案整理。前两节课把文件登记好、按章节拆分好了，现在有人来查资料。

AI 给出了一个答案，客户接着问："这个结论是从哪份报告的哪一页得出的？让我看看原文。"

如果你的档案系统只存了抄写的文字，没有记录来源，这时候就傻眼了。你翻遍系统也找不到原始出处，客户对整个系统的信任瞬间归零。

正确的做法是：每份抄录都贴上标签，写明"来自某某报告第 15 页第 3 段，坐标在左上角到右下角"。客户点一下就能跳到原文高亮处，看到证据，信任才建立得起来。

这就是证据链的价值：回答问题只是起点，能让客户回到原文验证才是及格线。

## 关键概念

| 概念 | 大白话解释 |
| :--- | :--- |
| **证据链（Evidence Chain）** | 从 AI 的回答一路追溯到原始文档具体位置的完整链条。链条断了，客户就没法验证答案对不对 |
| **证据等级（5 Levels）** | 证据能力分 5 档：L0 只给答案没来源、L1 给文档名、L2 给页码和章节、L3 给页码加坐标能高亮原文、L4 精确到字符级别。To B 生产至少 L3 起步 |
| **证据锚点（EvidenceAnchor）** | 每个内容块上贴的三组标签：身份信息（哪份文件、哪个版本）、定位信息（页码、坐标）、上下文信息（所属章节、检索方式）。缺一组就断链 |
| **Citations API** | Anthropic 提供的功能，AI 回答时自动标注"这句话引用了第几份文档的哪段原文"，精确到字符级别，幻觉率降低 50% |
| **端到端无损传递** | 证据信息从解析、切片、入库、检索、生成这 5 步全程不丢失。90% 团队在入库这一步为了性能把定位信息砍掉了 |
| **release_id（发布版本号）** | 记录这次回答使用的是哪个版本的索引和模型。客户三个月后问"当时你为什么这么答"，靠这个号就能还原现场 |

## 避坑提醒

- **以为存了来源信息就够了**：存了但 API 不返回给前端，客户看到的还是没来源的答案。证据链要从后端到前端全程贯通。
- **以为入库时砍掉坐标信息能提升性能**：为省 10% 内存把页码、坐标等元数据删了，上线后客户投诉"答案不可信"。2026 年主流向量数据库（pgvector、Milvus）都支持元数据和向量一起存，没必要砍。
- **以为评测只看答案准不准**：如果评测不验证引用是否正确，上线后可能 30% 的引用指向了错误的文档段落。引用错了比没有引用更危险，因为客户会以为你验证过了。
- **以为证据链是锦上添花**：2025 年起 To B 的 RAG 系统，证据链已经是合规基线。金融、法律、医疗行业必须做到字符级引用（L4），欧盟 AI 法案要求 AI 系统具备可追溯性。

## 一句话总结

> **AI 给答案只是开始，能让客户点一下就看到原文出处，才算合格的生产系统。**
