# 导学 05：音频视频来了，要从头造一套新管道吗？

## 一句话理解

音频、视频、图片本质上也是非结构化数据，前四节课的处理范式（解析、切片、证据链、质检）换个物理载体就能直接套用。

## 用生活打比方

继续我们的图书馆档案整理。前面四节课都在处理纸质文件（PDF），现在领导说："仓库里还有一批录音带、录像带和照片，也要归档进图书馆。"

你的第一反应可能是"这完全是另一种东西，得重新搞一套流程"。但仔细想想，其实归档的逻辑是一样的：录音需要先转成文字（相当于抽取内容），按说话人和话题拆分（相当于切片），标注"第 3 分 20 秒张三说的"（相当于证据锚点），检查转写质量（相当于质检）。

区别只在物理载体：纸质文件的定位靠"第 15 页左上角"，录音的定位靠"第 3 分 20 秒"，录像的定位靠"第 5 分钟的画面 + 第 5 分钟的声音"。底层的归档规范完全可以复用。

## 关键概念

| 概念 | 大白话解释 |
| :--- | :--- |
| **三轨道模型（Three-Track）** | 处理视频时分三条线独立跑：音轨（声音转文字）、视频轨（抽关键帧生成描述）、时间轴（用时间戳把三条线对齐）。独立处理再合并，比混在一起跑稳定 5 倍 |
| **说话人分离（Diarization）** | 多人对话时自动识别"谁在什么时候说了什么"。不做这一步，会议录音转出来就是一长段混在一起的文字，没法追溯是谁说的 |
| **时间戳对齐（segment_ts）** | 音频的时间戳就像 PDF 的页码。有了它，用户点击 AI 回答里的引用就能跳转到录音的精确时刻，和 PDF 高亮原文是同一个体验 |
| **视觉语言模型（VLM）** | 能同时理解图片和文字的 AI 模型，比如给它一张财报截图，它能读懂里面的表格内容。2026 年 90% 的图像任务用一个 VLM 调 API 就够了 |
| **同构思维** | 遇到新的数据类型时，先问"它的解析、切片、证据、质检分别对应什么"，会发现 90% 的工作量可以复用已有的框架 |
| **CLIP 嵌入** | 让图片和文字进入同一个搜索空间，这样"用文字搜图片"或"用图片搜文字"都能工作。比如输入"一只猫在花园里"就能搜到对应的照片 |

## 避坑提醒

- **以为处理 PDF 的方法不能用在音视频上**：音频的时间段就像 PDF 的页码，视频的关键帧就像合同的坐标。解析、切片、证据、质检这套范式直接套用，只需要把物理载体对应过去。
- **以为所有 PDF 都应该截图给 VLM 处理**：VLM 处理 PDF 的成本远高于传统 IDP 工具。正确做法是绝大部分用 IDP 工具处理，只有少量布局极端复杂的疑难件才走 VLM 兜底。
- **以为音频不用做说话人分离**：多人会议的录音如果不区分谁在说话，转出来的文字混在一起，客户问"谁提出了这个方案"根本没法追溯。说话人分离是生产环境的必选项。
- **以为可以跳过切片直接把整段录音文字存进去**：一段 1 小时的录音转成几万字的文字，不切片的话检索召回质量极差。按说话人、按话题、按场景切分，检索时才能精准找到需要的那一段。

## 一句话总结

> **音视频只是换了物理载体，解析、切片、证据、质检的范式完全一样，别被"多模态"三个字吓到。**
