# 导学 06：整条链路跑一遍，从散乱文件到可查询的数据资产

## 一句话理解

实践课把前五节的概念串成一条完整的处理链路，从资产清单登记到入库验证，跑通端到端的全流程。

## 用生活打比方

继续我们的图书馆档案整理。前五节课分别学了怎么登记格式、怎么拆分章节、怎么贴来源标签、怎么做入库质检、怎么处理录音录像。这节课是实操演练：真正走一遍完整的归档流程。

你拿到一批真实的档案（PDF、图片、音视频），从头开始：先填写入库清单、登记每份档案的编号和类型，然后逐一验证档案完整性，接着根据类型分别处理（纸质文件走 OCR 通道、录音走转写通道），处理完做质检，最后正式入库上架。

入库之后还要验证一下：随便挑几条记录，看看能不能从 AI 的回答追溯到原始档案的具体位置。能追溯到，说明整条链路通了；追溯不到，说明某个环节掉了链子。

## 关键概念

| 概念 | 大白话解释 |
| :--- | :--- |
| **资产清单（Manifest）** | 一份登记表，记录所有要处理的文件：每个文件有唯一编号（source_id）、文件类型、校验码（确保文件没被篡改）、是否允许进入索引等信息。整条链路的起点 |
| **源加载器（Source Loader）** | 负责读取文件并验证：确认文件存在、计算指纹、检查和清单登记的是否一致。它只做验证，不做业务处理 |
| **质量门禁（Quality Gate）** | 入口处的检查关卡，校验不通过的文件直接拦住，不让脏数据流入后续环节。预防成本远低于事后清理 |
| **解析适配器（Parser Adapter）** | 根据文件类型自动选择处理方式：PDF 走标记解析、图片走 OCR、音视频走转录。不需要手动指定，系统按清单里的类型字段自动路由 |
| **运行包裹（Parcel Run）** | 把清单验证、解析、切块、锚点、质检这些步骤串成一次完整的运行。设置好参数（解析方式、切块策略、发布版本号）后一键执行 |
| **契约测试** | 验证处理结果是否满足下游要求的测试，比如"每个切片必须有证据锚点""PDF 必须有页面坐标"。问题在测试阶段暴露，比上线后发现便宜得多 |
| **release_id（发布版本号）** | 给每次运行绑定一个版本号，后续追踪"这批数据是哪次处理产出的"。出了问题可以精确定位是哪次运行引入的 |

## 避坑提醒

- **以为跑通 Demo 就能上线**：Demo 阶段用几个文件测试感觉很顺畅，但生产环境的文件类型多样、质量参差不齐。必须用契约测试覆盖各种边界情况，比如缺少锚点、校验码不匹配等。
- **以为直接入库没风险**：去掉 dry-run 参数后数据会真实写入数据库。先用 dry-run 模式生成报告看效果，确认无误再正式入库。清理脏数据的成本远高于多跑一次预览。
- **以为链路只能处理固定路径的文件**：实际链路支持自定义输入和本地文件，不依赖固定的目录结构。你可以指定任意一个本地文件作为输入，走完整条处理流程。
- **以为非结构化数据和结构化数据是两套体系**：底层范式一致，契约验证、质量门禁、版本绑定这些机制都可以复用。第二周到第六周打下的结构化数据基础，80% 直接适用于非结构化数据处理。

## 一句话总结

> **整条链路从资产清单到入库验证跑通，处理后的数据就是下周 RAG 服务化的直接原料。**
